This modification is developed for below requirement:

When the item received exists on multiple POs, user is expected to receive a message to aware that, once got this message,
User has 2 options:
1. F1 back to previous UNDIR_IDENTIFY screen to scan correct PO in 'Rcv ID:' field or user F2 to lookup POs.
2. Enter again to continue receiving the pallet on current PO.

The message for user to see is:
"Multiple POs found for item, correct PO if needed, current PO is ^cur_po_num^."

This is developped for version 2018.1.2.3
